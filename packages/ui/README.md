# @odasy/ui

> The Odasy Design System — tokens + components shared across web and mobile.

## Architecture

```
packages/ui/
├── src/
│   ├── theme/                Source of truth (TypeScript)
│   │   ├── palette.ts        Raw colour values
│   │   ├── tokens.ts         Semantic tokens (light + dark)
│   │   ├── typography.ts     Fonts, scale, tracking, leading
│   │   ├── spacing.ts        4px-base scale + container widths
│   │   ├── radii.ts
│   │   ├── shadows.ts
│   │   ├── motion.ts         Easings, durations, named keyframes
│   │   ├── legacy.ts         Nightfall & Volt (legacy `/` landing only)
│   │   └── index.ts
│   ├── web/                  React + Tailwind components
│   │   ├── components/
│   │   ├── styles/
│   │   │   ├── tokens.generated.css   ← autogenerated
│   │   │   └── utilities.css
│   │   └── theme/            ThemeProvider + useTheme
│   ├── native/               React Native components
│   │   ├── components/
│   │   └── theme/
│   └── shared/
│       ├── utils/cn.ts
│       └── hooks/use-reduced-motion.ts
├── scripts/
│   └── generate-css-tokens.ts   tokens.ts → tokens.generated.css
└── tamagui.config.ts             (legacy bridge)
```

## How to consume

### Web

```ts
// Tokens (TS objects)
import { tokens, motion, typography } from '@odasy/ui/theme';

// Generated CSS — import once in your global stylesheet
import '@odasy/ui/styles/tokens.css';
import '@odasy/ui/styles/utilities.css';

// Components
import { Button, Card, MetaLabel, Stamp } from '@odasy/ui/web';
import { ThemeProvider, useTheme } from '@odasy/ui/web';
```

### Mobile (React Native)

```ts
import { tokens } from '@odasy/ui/theme';
import { Button, Card, Input, useTheme, ThemeProvider } from '@odasy/ui/native';
```

## Tokens

Tokens are declared in `src/theme/` as TypeScript objects. Web consumes them as
CSS variables (auto-generated by `pnpm --filter @odasy/ui gen:tokens`); mobile
imports them directly via `useTheme().tokens`.

The semantic layer has these top-level keys:

```
bg          surface levels: canvas, surface, elevated, muted
ink         text hierarchy: primary, secondary, tertiary, inverse, onAccent
border      hairlines:      default, strong, subtle
accent      primary brand:  base, action, hover, deep, soft, onSurface
warm        amber accent:   base, soft, ink
nature      moss accent:    base, soft, ink
info        blue accent:    base, soft
status      success / warning / danger / info
stamp       bronze / silver / gold / platinum / diamond / ink
selection   { bg, fg }
overlay     { scrim, glow }
```

Each key resolves to different values in `tokens.light` vs `tokens.dark`.

## Theming

Web: wrap your tree with `<ThemeProvider>` from `@odasy/ui/web`. The provider
sets the `ds-dark` class on `<html>` when dark mode is active. Tokens then
resolve to their dark values everywhere.

Mobile: wrap with `<ThemeProvider>` from `@odasy/ui/native`. Pass
`defaultTheme="system"` (default) to follow the OS appearance.

## Adding a token

1. If it's a brand-new colour, add the literal value to `palette.ts`.
2. Map it in `tokens.ts` (both light and dark).
3. Run `pnpm --filter @odasy/ui gen:tokens` to regenerate the CSS.
4. Use it in components via `var(--color-<key>)` (web) or
   `useTheme().tokens.<path>` (native).

## Adding a component

1. Drop a file in `src/web/components/` or `src/native/components/`.
2. Pull tokens via `var(--color-*)` (web) or `useTheme()` (native) — no
   hard-coded colours.
3. Export from the corresponding `index.ts` barrel.
4. Add a demo to `apps/web/app/ds/components/page.tsx`.

## Naming conventions

- Components: clean nouns. `Button`, `Card`, `Stamp`, `MissionCard`. Aurora is
  the **theme** of the DS, not a prefix on every component.
- Token keys: semantic intent, never visual. `accent.action` not `violet600`.
- CSS classes: `ds-` prefix for utility classes (`ds-card`, `ds-paper`,
  `ds-animate-*`).

## Showcase

Run `pnpm --filter @odasy/web dev` and visit `/ds` to browse tokens and
components live. The showcase is theme-toggleable and serves as the source of
visual truth for QA.
